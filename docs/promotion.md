## Code Context MCP — 让 AI 编码助手真正理解你的代码库

### 项目介绍

Code Context MCP 是一款开源的 MCP 服务器，使用 Go 语言实现，通过 stdio JSON-RPC 2.0 协议为 AI 编码助手提供代码语义搜索与结构分析能力。它将向量语义搜索、精确符号查找和影响范围分析整合为一套轻量级服务，帮助 AI 助手在处理中大型代码库时显著减少上下文消耗，提高回答质量。

项目地址：https://github.com/handy-h/code-context-mcp

---

### 我们试图解决的问题

在使用 AI 编码助手（如 Cursor、GitHub Copilot、Claude Desktop 等）辅助开发的过程中，以下几个痛点普遍存在：

**上下文窗口浪费严重。** AI 助手为理解代码往往需要反复调用 Read、Grep、Glob 等工具读取大量文件，每次都消耗宝贵的上下文窗口。对于拥有数百个文件的项目，助手可能在收集足够上下文之前就已耗尽上下文额度，导致回答质量下降甚至丢失前文信息。

**代码搜索停留在文本层面。** 现有的代码搜索方式主要依赖关键词匹配或正则表达式，无法理解搜索意图。当开发者想查找"处理用户认证的逻辑"或"数据库连接池初始化的地方"时，需要自己将意图拆解为具体的函数名或关键词，效率有限。

**重构时缺乏全局视野。** 执行函数重命名、接口修改等操作时，AI 助手很难一次性准确识别所有受影响的文件和调用点，容易遗漏引用导致编译错误或运行时故障。

**外部依赖过重。** 许多增强上下文的方案要求搭建外部向量数据库、部署复杂的 RAG 管线，对个人开发者和小团队而言门槛偏高。

### 核心能力

Code Context MCP 提供五个 MCP 工具，覆盖从搜索到分析的完整链路。

**语义代码搜索（code_search）。** 使用自然语言描述搜索意图即可定位相关代码片段。例如输入"处理 JWT 令牌验证的中间件"，系统通过向量相似度匹配返回最相关的函数或代码块，附带文件路径和相似度分数。这使 AI 助手无需猜测函数名即可快速找到目标代码。

**精确符号查找（symbol_search）。** 按函数名、变量名、类名等符号精确定位所有定义和引用位置，基于内存倒排索引实现毫秒级响应。支持驼峰与下划线命名的自动互转——搜索 `DiagnosisNotes` 会同时匹配 `diagnosis_notes`，覆盖 Go、Python 等不同命名风格的语言。

**影响范围分析（impact_analysis）。** 在执行删除、重命名或修改函数签名之前，一次调用即可获取所有受影响文件和代码位置的完整清单，并为每个位置生成具体的修改建议（区分定义、引用和 API 载荷），有效防止重构遗漏。

**文件结构摘要（file_context）。** 提供两种模式：full 模式返回完整文件内容，summary 模式返回文件的函数列表、类型定义和导入依赖等结构化信息。AI 助手可先通过 summary 概览文件结构，再精确读取所需部分，大幅节省上下文开销。

**手动索引构建（index_project）。** 支持按需触发全量索引，同时系统也内置了基于文件修改时间的增量更新机制——搜索时发现索引过期会自动在后台增量同步，无需手动干预。

### 技术优势

**语法结构感知的代码切分。** 区别于固定 token 窗口的粗暴切分，我们为 Go、Python、TypeScript/JavaScript、Rust、Vue 和 Markdown 七种语言分别实现了基于语法边界的切分策略。代码按函数、结构体、类、接口等语义单元自然分割，搜索结果天然语义完整，不会从函数体中间截断。对于不支持的语言，则降级为按行边界的固定窗口切分，保证通用性。

**双索引架构。** 语义向量索引负责模糊意图匹配（"错误处理的统一入口"），符号倒排索引负责精确定位（`HandleError` 在哪里被引用）。两者互补，覆盖了开发者搜索代码时的两类核心场景。

**零依赖本地模式。** 默认使用本地 JSONL 文件存储向量数据，不需要运行任何外部数据库。搭配 Ollama 本地嵌入模型，可以实现完全离线的端到端工作流，对个人项目和中小型代码库尤为友好。对于大规模代码库，也可切换至 Zilliz Cloud 后端。

**灵活的嵌入模型支持。** 提供 Ollama、OpenAI、Gemini 三种嵌入服务实现，通过环境变量切换，支持自定义模型名和向量维度。内置指数退避重试机制，对瞬态网络错误和速率限制自动恢复。

**增量索引更新。** 通过文件修改时间指纹检测变更，仅对变更文件重新索引，避免每次全量重建。增量更新失败时自动降级为全量重建，保证索引一致性。

**严格的安全措施。** 文件访问实现了完整的目录遍历防护，包括 `..` 路径检测、符号链接解析和路径范围验证，确保 AI 助手只能在项目目录内访问文件。

### 兼容性

Code Context MCP 基于标准 MCP 协议（版本 2024-11-05）的 stdio 模式实现，可与支持 MCP 的 AI 编码工具直接集成，目前已验证兼容的工具链包括 OpenCode、Cursor、Windsurf、Claude Desktop、GitHub Copilot 和华为云 CodeArts（码道）。项目提供 Linux、macOS 和 Windows 的预编译二进制，也提供多架构 Docker 镜像。

### 快速开始

```bash
# 克隆并构建
git clone https://github.com/handy-h/code-context-mcp.git
cd code-context-mcp
make build

# 部署（构建并复制到 code-context-mcp/ 目录）
make deploy

# 运行测试
make test
```

配置通过环境变量完成，最简配置只需设置嵌入模型服务和项目路径即可启动。详细的配置说明请参阅项目仓库中的文档。
